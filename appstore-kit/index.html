REC 01 · REAL BUG

Sandbox kills gitignore UX it was designed to keep

GitService.findRepoRoot shells out to git rev-parse, which the sandbox guard disables — so in the App Store build repoRoot() is always nil, GitInfo.isRepo is always false, the sandbox guard inside info() is dead code, and the “add to .gitignore” checkbox in New File never appears even though it's plain file I/O that works sandboxed. Fix with filesystem-based detection:

// GitService.findRepoRoot — sandbox-safe repo detection
if AppSandbox.isActive {
    var dir = folder.standardizedFileURL
    while dir.path != "/" {
        // .git is a dir in normal checkouts, a file in worktrees
        if FileManager.default.fileExists(atPath: dir.appending(path: ".git").path) { return dir }
        dir = dir.deletingLastPathComponent()
    }
    return nil
}
// …existing rev-parse path for the unsandboxed edition
REC 02

Dual-license note for the GPL

You own 100% of the copyright, so App Store distribution is legally yours to grant — add one line to the README/LICENSE stating that official binaries on the App Store are additionally offered under Apple's standard EULA. Prevents the classic “GPL app pulled from the store” drama (VLC, 2011).

REC 03

Deployment target macOS 26.5 shrinks the audience

Only Macs on the very latest OS can install. Deliberate choice given the SwiftUI APIs you use — just know the store page will show “Requires macOS 26.5 or later.”

REC 04

Description below is written for the sandboxed edition

It deliberately omits the git-leak guard, whole-machine scanning, and the bundled CLI — Developer ID-only features. Don't paste marketing copy from the README, which advertises all three.

REC 05

After merging, update the live Homebrew formula

This branch moved the CLI to cli/ — the tap formula needs --package-path cli when app-store-prep merges to main, or brew install breaks.

App Review Notes
EnvHub is a fully offline developer utility for organizing and editing .env (dotenv) configuration files. There are no accounts and no server component, so no demo credentials are required. How to test: create any folder containing a file named ".env" with a few KEY=value lines, then click "Add Project…" in EnvHub and select that folder. The app runs in the App Sandbox; folder access is granted through the standard open panel and persisted with security-scoped bookmarks. All "secret" values visible in the app and screenshots are fabricated demo data, and the app masks values by default. Encryption: the optional export feature encrypts files locally using Apple's CryptoKit (AES-256-GCM, scrypt key derivation). The app makes no network connections of any kind, and ITSAppUsesNonExemptEncryption is set to NO accordingly. EnvHub is open source; the developer holds all rights and additionally licenses App Store binaries under Apple's standard EULA. Source: github.com/cs4alhaider/EnvHub

# 05 · SUBMISSION · FROM THIS BRANCH TO “WAITING FOR REVIEW”

The path to submit, in nine steps.

01

Fix the project (Blockers 01–04)

In Xcode: add the AppStore configuration with sandbox + entitlements + team, bump versions to 1.0.0, set copyright and ITSAppUsesNonExemptEncryption. Verify PrivacyInfo.xcprivacy shows under the EnvHub target (it was added to the synced folder today — it should appear automatically).

02

Developer portal: IDs & capabilities

Register bundle ID net.alhaider.EnvHub with the App Groups capability and attach the registered group group.net.alhaider.EnvHub. The group itself, the code constant, and both entitlements files already match (Blocker 03 — resolved).

03

Publish the privacy policy

Commit PRIVACY.md (repo root, drafted) and push to main. Confirm the URL renders in a logged-out browser.

04

Create the ASC app record

App Store Connect → My Apps → “+” → New App → macOS, name EnvHub, bundle ID, SKU envhub-mac-001, English (or your primary locale).

05

Archive & upload

Xcode → Product → Archive (AppStore configuration) → Organizer → Distribute App → App Store Connect → Upload. Automatic signing with the ADP team creates the provisioning profile with the app group. Wait for processing (~10 min).

06

Fill the version page

Paste name, subtitle, promo text, description, keywords, URLs and What's New from #metadata.env. Upload the 8 screenshots in order. Set copyright.

07

App Privacy + Pricing

Privacy questionnaire → “Data Not Collected”. Pricing → Free, all territories. Age rating → 4+.

08

Attach the build + review notes

Select the processed build, paste the App Review Notes, and answer export compliance if asked (standard/exempt encryption only).

09

Submit for Review

First macOS reviews typically take 1–3 days. If rejected, it's almost always a question, not a verdict — answer in Resolution Center and resubmit.
